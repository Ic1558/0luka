# 0luka Constitution: Gateway Architecture (v1.1)

## 1. The Core Philosophy: Capability-Centric Authority
อำนาจในการปฏิบัติการ (Execution Authority) ภายในระบบ 0luka ไม่ได้ผูกติดกับตัวตนของโมเดลหรือเอเย่นต์ใดๆ แต่จะมอบให้แก่กลไก (Mechanism) ที่ปฏิบัติงานผ่านเส้นทางที่ได้รับอนุญาตเท่านั้น

## 2. The Truth Chain (เส้นทางแห่งความจริง)
การเปลี่ยนแปลงใดๆ ต่อระบบต้องไหลผ่านเส้นทางเดียว (Unidirectional Flow):
1. **Strategist (Human + GM/GG):** ผู้ออกแบบสเปคและอนุมัติ (Think / Spec / Approve)
2. **Orchestrator (Any Capable Model):** ผู้เรียบเรียงแผนงานเข้าสู่ Workspace (Compile / Plan)
3. **Bridges (Antigravity / OpenCode / CLI):** ช่องทางเชื่อมต่อที่ได้รับอนุญาตสู่ระบบไฟล์ (Execution Pathway)
4. **Kernel (Core):** ปลายทางที่เปลี่ยนแปลงได้เฉพาะผ่าน Artifacts + PRP + Promotion

## 3. Standards of Conduct
- **Evidence-Based:** ทุกงานต้องมีหลักฐาน (Artifacts/Logs) กำกับเสมอ
- **Deterministic:** ผลลัพธ์ต้องคาดเดาได้และคงเส้นคงวา
- **Self-Correcting:** ระบบต้องสามารถตรวจจับและแก้ไขข้อผิดพลาดได้ด้วยตนเอง
