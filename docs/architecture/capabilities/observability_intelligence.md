# Observability Intelligence Capability

## Layer

Observability Layer

## Purpose

Synthesize observability signals into higher-level insights while remaining read-only.

## Scope

- Telemetry aggregation and interpretation
- Anomaly detection and reporting
- Narrative and summary generation

## Non-Goals

- Execution or remediation
- Policy changes
- Runtime mutation

## Interfaces

- Inputs: logs, runtime signals, decision artifacts
- Outputs: reports, summaries, insight artifacts

## Ownership

- Canonical owner: this document
- Execution owner: observability tooling and reporting services
- Change authority: docs/architecture/0LUKA_ARCHITECTURE_DECISION_RECORDS.md
