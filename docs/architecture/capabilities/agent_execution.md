# Agent Execution Capability

## Layer

System / Services Layer

## Purpose

Coordinate agent workflows using the runtime substrate and policy constraints.

## Scope

- Task routing and agent lifecycle management
- Queue orchestration within approved governance
- Emitting agent run records and outcomes

## Non-Goals

- Defining policy governance
- Bypassing runtime constraints
- Autonomous control-plane behavior

## Interfaces

- Inputs: approved tasks, policy constraints, runtime status
- Outputs: agent run records, execution summaries

## Ownership

- Canonical owner: this document
- Execution owner: system services coordinating agents
